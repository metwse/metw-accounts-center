1//! # metw-accounts-center
2//!
3//! metw-accounts-center is an identity management system implemented as a
4//! zero-knowledge service.
5//!
6//! Goals:
7//! - Authentication Service: An application can use this service as an
8//!   authentication authority.
9//! - End-to-End Encryption (E2EE): Users have a public-private key pair
10//!   and a master key. Their master keys and private keys are stored on the
11//!   server, encrypted client-side.
12//!
13//! Non-Goals:
14//! - This microservice must not be extended with non-authentication features:
15//!   Profiles, posts, shares, and similar concepts are out of scope for this
16//!   service.
17//!
18//!
19//! ## Architecture: E2EE-Authentication
20//!
21//! Account public keys are served publicly. When users log into their accounts,
22//! they retrieve their encrypted private keys from the server and decrypt them
23//! locally. This private key is then used for OAuth2-like flows, where the
24//! users directly authorize other applications.
25//!
26//! ```txt
27//! +-- CLIENT - (Account Creation) --------------------------------+
28//! |                                                               |
29//! |     User credentials                      Key encryption key  |
30//! |    +------------------+  Key stretching  +-----+              |
31//! |  --- password ---------------------------> KEK |              |
32//! |  | |                  |                  +--|--+              |
33//! |  | | client generated |                     | Encrypt         |
34//! |  | | random keys:     |                  +--v-----------+     |
35//! |  | | - master key -----------------------> encrypted MK |     |
36//! |  | | - private key ----------------------> encrypted PK |     |
37//! |  | | - public key ---------------------  +--------------+     |
38//! |  | +------------------+               |         |             |
39//! |  |                                    |         |             |
40//! |  |  Argon2  +----------------------+  |         |             |
41//! |  -----------> client password hash |  |         |             |
42//! |             +----------------------+  |         |             |
43//! |                       |               |         |             |
44//! +-----------------------|---------------|---------|-------------+
45//!                         |               |         |
46//! +-- SERVER -------------|---------------|---------|-------------+
47//! |                       |               |         |             |
48//! |                 Authentication   Public Key   Key             |
49//! |                 Storage          Registry     Vault           |
50//! |                                                               |
51//! +---------------------------------------------------------------+
52//! ```
53//!

55//! # Setup
56//!
57//! Required environment variables for repository and client setup are listed
58//! in fields of [`Config`](state::Config) struct.
59//!
60//! Once you have prepared the `.env` file, run the database migrations via:
61//! ```sh
62//! cd state/
63//!
64//! cargo sqlx migrate run
65//! ```
66//! (See the related
67//! [sqlx-cli](https://github.com/transact-rs/sqlx/tree/main/sqlx-cli#create-and-run-migrations)
68//! documentation)
69//!
70//! > **Important:** To ensure application security, carefully read the
71//! > [Setup Recommendations](state#setup-recommendations)
72//! > section.
